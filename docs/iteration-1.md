# Звіт за ітерацію №1 (Lab 34) — Проєкт "RealEstateAgency"

## 1. Досягнуті цілі ітерації
На першій ітерації було закладено архітектурний фундамент системи та реалізовано перший вертикальний зріз функціональності. Проєкт успішно розділено на шари згідно з Clean Architecture.

### Що реалізовано:
* **Сценарій:** Реєстрація та відображення об'єктів нерухомості (квартир та будинків) з автоматичним розрахунком комісії.
* **Domain:** Створено базову ієрархію класів `Property` та сутностей-спадкоємців. Впроваджено інваріанти (валідація ціни та адреси в конструкторі).
* **Application:** Створено сервіс `PropertyService`, що координує роботу між UI та сховищем.
* **Infrastructure:** Реалізовано `InMemoryRepository` для тимчасового зберігання даних.
* **Console UI:** Розроблено текстовий інтерфейс для демонстрації результатів.

## 2. Статус артефактів (згідно зі скріншотами)
Всі необхідні документи створені та розміщені в папці `docs/`:
* `vision.md` — концепція та сценарії.
* `backlog.md` — план розробки на майбутні лабораторні роботи.
* `class-diagram.md` — UML-схема структури класів.
* `TESTING.md` — опис пройдених юніт-тестів (зокрема валідації ціни).
* `DEMO.md` — скріншоти роботи консольної програми.

## 3. Технічні деталі
* **Поліморфізм:** Реалізовано через абстрактний метод `CalculateCommission()` (5% для квартир, 8% для будинків).
* **Обробка помилок:** Використано винятки `ArgumentException` для захисту доменної моделі від некоректних даних.
* **Тестування:** Пройдено юніт-тести, що підтверджують стабільність базової логіки.

## 4. План на наступну ітерацію (Lab 35)
1.  **Persistence:** Заміна InMemory сховища на роботу з реальними файлами (JSON серіалізація).
2.  **Запити:** Впровадження LINQ для пошуку об'єктів за параметрами.
3.  **Refactoring:** Подальше відокремлення логіки згідно з SOLID.
